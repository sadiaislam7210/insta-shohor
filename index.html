How javascript works on browser?

Javascript is a client site scripting language which is written inside script tag in html. When we open the html file in browser which have already some javascript code, that javascript code recieve by a browser engine. For chrome browser it is called v8 engine which convert javascript code into machine code line by line. Then that converted code sent to system for execute. By this we can see the output and that's how javascript works. V8 engine have a feature called parser which stop the execution if found any error in code.Another feature called AST which create data structure after parsing.

What does Javascript event loop do?

Event loop is a connector between queue and call stack.We know all the synchronous functions stay on call stack and if there is a asynchronous function browser engine pass it to queue. In call stack synchronous function stay as a stack which means last in and first out. By this all the synchronous function finish. Than it calls queue for the remaining asynchronous functions. Queue send the asynchronous functions through event loop which is works as a first in and first out.That means which asynchronous functions comes at first in queue goes first to call stack through event loop.

What's the difference between local storage and session storage?

In session storage data stays at temporary. Which means if we close the browser the session storage will be cleared. But in local storage data stays at permanent untill we clear them manually.But if we create a new tab session storage auto creates and if we close the tab it auto deletes. But in case of local storage it doesn't work for any specific tab. If we open same page on multiple tab , they will have a different session storage. On the other hand local storage will only have one in case of opening same page on multiple tab.
